# Модератор клана BB — Telegram-бот + SPA Dashboard

Бот для набора в игровой клан: проводит опрос с кнопками, ведёт базу
участников, синхронизирует статус с Telegram-группой и предоставляет
админский дашборд на React + Vite.

## Возможности

### Опрос для новичков (`/start`)

Порядок шагов:

1. **Возраст** — набор с 21 года
2. **Уровень** — от 100+
3. **Активность** — минимум раз в неделю
4. **Режим** — FPP / TPP / Mixed mode
5. **Ник в игре** — текстовый ввод (кнопка «Назад»)
6. **Имя** — текстовый ввод (кнопка «Назад»)
7. **Discord** — текстовый ввод или «—» (кнопка «Назад»)
8. **Финал** — кнопки на Telegram-группу и Discord

**Правила:**
- 2 попытки пройти опрос; после второго провала — автоматический чёрный список
- Пользователи из чёрного списка не могут начать опрос
- В группу допускаются только пользователи с завершённым опросом или уже состоящие в базе клана

### Админ-команды

| Команда | Описание |
|---------|----------|
| `/members` | Список участников: TG / игровой ник / Discord |
| `/stats` | Статистика FPP vs TPP |
| `/search <запрос>` | Поиск по никам и именам |
| `/blacklist` | Просмотр чёрного списка |
| `/unblacklist <id>` | Снять блокировку |
| `/kick_non_members` | Удалить из группы тех, кто не в базе клана |
| `/sync_group` | Принудительно синхронизировать фактический состав группы |
| `/assign_titles` | Проставить игровые ники как теги всем участникам |
| `/admin_help` | Справка по командам |

### Авто-теги участников

При вступлении пользователя в супергруппу бот автоматически назначает ему
**custom title** (бейдж рядом с именем), равный его **игровому нику** из опроса.

Требования:
- чат должен быть **супергруппой** — Telegram не поддерживает `custom_title`
  в каналах и обычных группах;
- бот должен быть **администратором** с правом **«Добавлять администраторов»**
  (`can_promote_members`);
- ник обрезается до **16 символов** — ограничение Telegram.

Для отображения тега Telegram требует минимальный admin-статус. Бот выдаёт
только `can_pin_messages` — без права приглашать по ссылке и без управления
настройками группы.

Команда `/assign_titles` позволяет проставить теги всем уже состоящим
в группе участникам, чьи ники сохранены в БД.

### Контроль доступа в группу

- **Join request** одобряется автоматически только для прошедших опрос или
  уже состоящих в базе клана
- Непрошедшие опрос кикаются при попытке входа (soft kick: ban+unban)
- Пользователи из **blacklist** получают **жёсткий бан** в Telegram — повторный
  вход по invite-ссылке из бота блокируется (без цикла join → remove)
- Soft kick за незавершённый опрос **не** добавляет в blacklist (можно пройти
  опрос и зайти позже)
- Добровольный выход (`LEFT`) **не** добавляет в blacklist
- Бан / кик админом добавляет в blacklist
- При старте бот синхронизирует Telegram-баны со всеми записями blacklist

### SPA Dashboard

Админский дашборд доступен по адресу `http://localhost:8080` (порт задаётся
`DASHBOARD_PORT`). Страницы:

- `/members` — список активных участников с фильтром, поиском, сортировкой и пагинацией
- `/blacklist` — чёрный список с причиной блокировки на русском и кнопкой «Вернуть доступ»
- `/inactive` — игроки, у которых последняя катка была 7+ дней назад (168+ часов)

При разблокировке из дашборда:
- если причина «не прошёл опрос» — пользователю отправляется сообщение пройти `/start` заново
- в остальных случаях — попытка вернуть в группу (approve join request или invite-ссылка)

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# заполните .env

alembic upgrade head

# Запуск бота (Telegram polling)
python -m bot.main

# В отдельном терминале — запуск API дашборда
python -m dashboard.backend.run
```

## Сборка фронтенда

```bash
cd dashboard/frontend
npm install
npm run build
```

После сборки SPA раздаётся самим FastAPI-сервером из `dashboard/frontend/dist`.

## Безопасность дашборда

Если `DASHBOARD_API_KEY` задан, все `POST` endpoints (kick, unblock и т.д.)
требуют заголовок `X-API-Key`. В production фронтенд передаёт его через
`VITE_DASHBOARD_API_KEY`. Если ключ не задан, дашборд работает без аутентификации
(удобно для локальной разработки).

## Настройка

### 1. Telegram Bot

1. Создайте бота через [@BotFather](https://t.me/BotFather).
2. Скопируйте токен в `BOT_TOKEN`.
3. Узнайте свой Telegram ID (например, через
   [@userinfobot](https://t.me/userinfobot)) и добавьте в `ADMIN_IDS`.
4. Создайте **супергруппу** клана.
5. Добавьте бота **администратором** супергруппы с правами:
   - Удаление участников (`can_restrict_members`)
   - Блокировка пользователей
   - **Добавление администраторов** (`can_promote_members`) — обязательно для auto-тегов
6. Укажите `GROUP_ID` (для супергрупп начинается с `-100…`).
7. Укажите `TELEGRAM_GROUP_LINK` — статическая invite-ссылка для кнопки после опроса.

> **Важно:** для обработки `chat_member` и `chat_join_request` в `allowed_updates`
> бот должен получать соответствующие апдейты. Privacy Mode в ЛС для опроса
> не критичен, но для группы бот должен быть админом.

### 2. Переменные окружения

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `GROUP_ID` | ID супергруппы клана (со знаком `-100…`) |
| `ADMIN_IDS` | ID администраторов через запятую |
| `TELEGRAM_GROUP_LINK` | Invite-ссылка на супергруппу клана |
| `DISCORD_LINK` | Invite-ссылка Discord (опционально) |
| `DATABASE_PATH` | Путь к SQLite-файлу (по умолчанию `data/bot.db`) |
| `MAX_SURVEY_ATTEMPTS` | Максимальное число попыток опроса (по умолчанию 2) |
| `DASHBOARD_PORT` | Порт FastAPI дашборда (по умолчанию 8080) |
| `DASHBOARD_API_KEY` | API-ключ для защиты мутающих endpoints (опционально) |
| `GROUP_SYNC_INTERVAL_MINUTES` | Интервал фоновой сверки состава группы (по умолчанию 10) |

## Автоматический чёрный список

Пользователь попадает в чёрный список если:

- провалил опрос 2 раза (`survey_failed` / `survey_attempts_exhausted`);
- был забанен / кикнут админом (`removed_from_group`, `kicked_from_dashboard`);
- удалён командой `/kick_non_members`.

**Не** попадает в blacklist при добровольном выходе из группы.

Снять блокировку: `/unblacklist <telegram_user_id>` или кнопка в дашборде.

## Синхронизация с реальной группой

- **При старте бота** — импорт админов группы и участников с завершёнными анкетами
- **Событийная синхронизация** — `chat_member`, `chat_join_request`, service messages
- **Фоновая сверка** — каждые `GROUP_SYNC_INTERVAL_MINUTES` (по умолчанию 10)
- **Ручная** — `/sync_group`

## Проверка активности через op.gg

На странице неактивных игроков бот периодически проверяет профили на op.gg.
Игрок считается неактивным, если `last_match_at <= now - 168h`.

При **первом вступлении** в группу сразу выполняется проверка op.gg (дата последнего
матча и флаг `is_inactive`). Дальше в периодических джобах участники младше
**168 часов (7 дней)** в группе пропускаются (grace по `group_members.joined_at`).

## Статусы участников

- **Активен** — в `members`, не в `blacklist`, последняя катка < 168 часов назад
- **Удалён из клана** — в `members`, но отсутствует в `group_members`
- **Blacklist** — запись в `blacklist`
- **Неактивен** — последняя катка была 168+ часов назад

Для legacy-участников используется `is_legacy = 1` и дата `2001-01-01`.
